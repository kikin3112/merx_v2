# Committee: Product

> Strategic product alignment body ensuring all squads build toward the same vision.

---

## Identity

| Attribute | Value |
|-----------|-------|
| **Level** | L6 — Governance |
| **Frequency** | Weekly |
| **Duration** | 45 min |
| **Chair** | CPO |
| **Members** | CPO, Head of Product, Product Owners (one per squad) |

---

## Purpose

Align **product priorities** across squads. Review RICE scores. Resolve inter-squad priority conflicts. Track product OKRs.

---

## Agenda Template

```
1. [5 min]  Review action items from last meeting
2. [10 min] North Star metrics update
3. [15 min] Cross-squad priority review (RICE-scored backlog)
4. [10 min] Dependency conflicts / blockers
5. [5 min]  Decisions + action items
```

---

## Scope of Decisions

| Decides | Does NOT Decide |
|---------|----------------|
| Feature priority across squads | How features are built |
| Cross-squad dependency sequencing | Architecture choices |
| Roadmap adjustments | Hiring / staffing |
| Feature scope (MVP vs full) | Budget allocation |

---

## Outputs

| Output | Destination | Frequency |
|--------|------------|-----------|
| Updated cross-squad priority list | Squads | Weekly |
| Roadmap adjustment notes | CEO + All org | Monthly |
| Dependency resolution decisions | Affected squads | As needed |
| Product OKR scorecard | CEO | Quarterly |

---

## Rules

1. **RICE scores required** — no feature discussed without a RICE score
2. **Data over opinions** — bring user data, metrics, or customer feedback
3. **POs must attend** — represents their squad's interests and constraints
4. **Decisions are binding** — squads execute, don't re-debate
