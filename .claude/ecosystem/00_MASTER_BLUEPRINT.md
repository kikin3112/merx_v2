## How to Read This Ecosystem

1. **Start here** — This blueprint gives you the full map.
2. **System documents** (01-08) define the rules, culture, and mechanisms that apply to ALL nodes.
3. **Node manuals** define each agent's specific purpose, responsibilities, inputs/outputs, metrics, and operational rules.
4. **Every node operates under** the system-wide documents. Node manuals reference but DO NOT duplicate system rules.
5. **For AI agents**: Each node manual is self-contained enough to be loaded independently. Load the relevant system document only when cross-referencing is needed.

---

## Core Principles for Reading

- **Standalone nodes**: Each file can be consumed independently without loading the entire ecosystem.
- **Cross-references**: Use relative links. Never duplicate content — reference it.
- **Token efficiency**: Files are designed for minimal token consumption while maximizing information density.
- **Living system**: This ecosystem evolves. Version control tracks all changes.
