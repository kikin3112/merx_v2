# Committee: Security

> Strategic security governance body reviewing posture, compliance, and incident patterns.

---

## Identity

| Attribute | Value |
|-----------|-------|
| **Level** | L6 — Governance |
| **Frequency** | Monthly |
| **Duration** | 60 min |
| **Chair** | Head of Security (SecOps) |
| **Members** | Head SecOps, Security Engineer, VP Eng, Compliance Analyst |

---

## Purpose

Review **security posture**, address **vulnerability trends**, update **policies**, and plan **penetration testing** and **compliance activities**.

---

## Agenda Template

```
1. [10 min] Security metrics dashboard review
2. [15 min] Vulnerability findings since last meeting (scan results, pen tests)
3. [10 min] Incident review (if any)
4. [10 min] Compliance status update (DIAN, Habeas Data)
5. [10 min] Policy updates / pen test planning
6. [5 min]  Action items
```

---

## Outputs

| Output | Destination | Frequency |
|--------|------------|-----------|
| Security posture report | CEO + VP Eng | Monthly |
| Updated security policies | All engineering | As needed |
| Pen test schedule | DevOps + Squads | Quarterly |
| Compliance status | CEO | Monthly |
| Vulnerability remediation tracking | Squads | Monthly |

---

## Rules

1. **Confidentiality** — vulnerability details are need-to-know
2. **Remediation deadlines are mandatory** — critical < 48h, high < 1 week
3. **Pen test results reviewed in full** — no skipping findings
4. **Annual policy review** — all security policies reviewed at minimum annually
