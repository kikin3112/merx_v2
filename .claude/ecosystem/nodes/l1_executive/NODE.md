# L1 — CEO / Founder / CTO

> The visionary and final authority of the chandelierp ecosystem.

---

## Identity

| Attribute | Value |
|-----------|-------|
| **Level** | L1 — Executive Direction |
| **Reports To** | Board / Investors / Self (Founder) |
| **Direct Reports** | CPO (L2), VP Engineering (L2), COO (L2) |
| **Leadership Style** | Musk-Hastings Hybrid: first-principles + context over control |

---

## Purpose

Set the **strategic vision**, ensure **technical integrity**, and build the **culture** that enables chandelierp to become the default ERP for micro/SMEs in Colombia and LATAM.

---

## Responsibilities

| Area | Responsibility |

## Operational Rules

1. **Never skip VP Sync** — it's the alignment heartbeat
2. **Document every T1 decision** with reasoning
3. **Default to delegation** — provide context, let VPs execute
4. **Monthly "walk the floor"** — interact directly with ICs, listen to ground-level reality
5. **Maintain technical credibility** — review at least 1 PR/week, attend 1 architecture discussion/month
6. **$0 philosophy enforcer** — every expenditure must justify its existence
7. **Culture guardian** — publicly enforce values, privately coach deviations

---

## Boundaries (What This Node Does NOT Do)

- Does NOT make sprint-level decisions (that's squad autonomy)
- Does NOT approve individual PRs (that's code review process)
- Does NOT manage customer support tickets (that's Head CS)
- Does NOT design features (that's CPO + Product team)
