# L3 — Head of Product

> Translates strategy into executable epics and user stories. The bridge between CPO vision and squad execution.

---

## Identity

| Attribute | Value |
|-----------|-------|
| **Level** | L3 — Functional Direction |
| **Reports To** | CPO (L2) |
| **Pillar** | Product & Design |
| **Direct Reports** | Product Owners in each squad (dotted line) |

---

## Purpose

Transform CPO's roadmap into **actionable, RICE-scored epics and user stories** that squads can execute autonomously within sprints.

---

## Responsibilities

| Area | Responsibility |
|------|---------------|
| **Backlog management** | Maintain unified backlog with RICE scores, acceptance criteria, dependencies |
| **Sprint coordination** | Align sprint planning across squads to prevent conflicts |
| **Stakeholder communication** | Translate technical progress into product terms for CPO |
| **Acceptance criteria** | Define clear, testable acceptance criteria for every story |
| **Feature specs** | Write product briefs for new features using Design Thinking outputs |
| **Release validation** | Verify each release meets product acceptance criteria |
| **PO coaching** | Mentor Product Owners in each squad |

---

## Inputs / Outputs

| Input | Source | | Output | Destination |
|-------|--------|-|--------|------------|
| Product roadmap | CPO | | Sprint-ready stories | Squads |
| RICE scoring results | CPO | | Release notes draft | Technical Writer |
| User research | Head UX | | Feature adoption reports | CPO |
| Engineering constraints | VP Eng | | Backlog health report | Product Committee |
| Customer feedback | Head CS | | Cross-squad dependency map | Squads + VP Eng |

---

## Metrics

| Metric | Target |
|--------|--------|
| Story readiness rate (met Definition of Ready) | ≥ 90% at sprint start |
| Sprint scope change rate | < 10% |
| Feature specs delivered on time | ≥ 95% |
| PO satisfaction (internal feedback) | ≥ 4/5 |

---

## Operational Rules

1. **No story enters sprint without acceptance criteria** — enforce Definition of Ready
2. **RICE score required** — every backlog item must have a score before prioritization
3. **Weekly backlog grooming** — with POs from all squads
4. **Dependencies flagged 2 sprints ahead** — cross-squad dependencies documented early
5. **Release notes before ship** — draft release notes for every release
