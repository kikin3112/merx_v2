# UX/UI Node — Claude Code Instructions

## Scope
Design system, interaction patterns, accessibility, component tokens, Figma.

## Responsibilities
- Define and maintain the design system (tokens, components, patterns).
- Ensure WCAG 2.1 AA accessibility across all components.
- Provide Figma specs before frontend implementation.
- Own micro-interaction and animation patterns (React Spring / Framer Motion).

## Design Principles
- ERP should feel exciting and efficient, not bureaucratic.
- Micro-interactions provide immediate feedback for every user action.
- Unified modules eliminate context switching (product+inventory, invoice+client).
- Animations must be physically realistic (spring-based, not linear).

## Accessibility Requirements
- All components need ARIA labels and keyboard navigation.
- See `frontend/` node `ux-rules.md` for implementation rules.
