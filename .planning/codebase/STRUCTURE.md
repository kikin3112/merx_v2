## Directory Purposes

### Backend: `backend/app/`

**Core Application:**
- `main.py` - FastAPI app initialization, middleware setup, exception handlers, route registration
- `config.py` - Pydantic Settings; loads and validates all environment variables at startup

**Data Layer (`datos/`):**
- `db.py` - SQLAlchemy engine, session factory, RLS context management
- `modelos.py` - Core business entities: Productos, Ventas, VentasDetalles, Terceros, Usuarios, Inventarios, etc.
- `modelos_tenant.py` - Multi-tenant infrastructure: Tenants, Planes, Suscripciones, UsuariosTenants, HistorialPagos
- `modelos_crm.py` - CRM domain: Contactos, Deals, Etapas, ActivityLog
- `modelos_pqrs.py` - Support system: TicketsPQRS, RespuestasPQRS
- `modelos_calificaciones.py` - Supplier/customer ratings: Calificaciones
- `esquemas.py` - Pydantic DTOs for all request/response validation (grouped by entity)
- `mixins.py` - Reusable model fields: timestamps, audit fields, soft deletes
- `audit_listeners.py` - SQLAlchemy event listeners for automatic audit trail logging

**Routes (`rutas/`):**
One file per domain, each with a FastAPI APIRouter instance:
- `auth.py` - /api/v1/auth/* (login, refresh, token exchange, password change)
- `tenants.py` - /api/v1/tenants/* (CRUD, registration, subscription management)
- `usuarios.py` - /api/v1/usuarios/* (user management, roles)
- `productos.py` - /api/v1/productos/* (product catalog CRUD)
- `terceros.py` - /api/v1/terceros/* (suppliers, customers, partners)
- `ventas.py` - /api/v1/ventas/* (sales orders, invoicing)
- `compras.py` - /api/v1/compras/* (purchase orders, receiving)
- `inventarios.py` - /api/v1/inventarios/* (stock movements, adjustments)
- `cotizaciones.py` - /api/v1/cotizaciones/* (quotes)
- `contabilidad.py` - /api/v1/contabilidad/* (journal entries, trial balance)
- `cuentas_contables.py` - /api/v1/cuentas-contables/* (chart of accounts)
- `configuracion_contable.py` - /api/v1/contabilidad/configuracion/* (accounting settings)
- `periodos_contables.py` - /api/v1/contabilidad/periodos/* (fiscal periods)
- `facturas.py` - /api/v1/facturas/* (invoices, electronic invoicing)
- `cartera.py` - /api/v1/cartera/* (receivables, collections)
- `medios_pago.py` - /api/v1/medios-pago/* (payment methods)
- `crm.py` - /api/v1/* (CRM operations: deals, contacts, activity)
- `pqrs.py` - /api/v1/pqrs/* (support tickets, complaints)
- `calificaciones.py` - /api/v1/calificaciones/* (ratings)
- `sse.py` - /api/v1/sse/* (real-time updates via Server-Sent Events)
- `reportes.py` - /api/v1/reportes/* (business intelligence)
- `ordenes_produccion.py` - /api/v1/ordenes-produccion/* (production orders)
- `recetas.py` - /api/v1/recetas/* (recipes for manufacturing)
- `health.py` - / and /health/* (health checks, no auth required)

**Services (`servicios/`):**
Domain logic encapsulated in service classes:
- `servicio_tenants.py` - ServicioTenants: tenant CRUD, onboarding, user assignment
- `servicio_productos.py` - ServicioProductos: product catalog, pricing, bulk operations
- `servicio_ventas.py` - ServicioVentas: sales order creation, invoicing, payment posting
- `servicio_inventario.py` - ServicioInventario: stock transactions, movements, adjustments
- `servicio_contabilidad.py` - ServicioContabilidad: journal entries, postings, trial balance
- `servicio_crm.py` - ServicioCRM: deals, activities, contact management
- `servicio_pqrs.py` - ServicioPQRS: ticket creation, assignment, resolution
- `servicio_calificaciones.py` - ServicioCalificaciones: ratings, reviews
- `servicio_audit.py` - ServicioAuditLog: audit trail recording
- `servicio_sse.py` - SSEManager: real-time event broadcasting to clients
- `servicio_almacenamiento.py` - ServicioAlmacenamiento: file uploads (S3/R2)
- `servicio_pdf.py` - ServicioPDF: PDF generation (invoices, reports)

**Middleware (`middleware/`):**
Request processing pipeline:
- `tenant_context.py` - Extracts X-Tenant-ID header, establishes RLS context, blocks maintenance mode writes
- `user_context.py` - Extracts user_id from JWT, establishes ContextVar for audit

**Utilities (`utils/`):**
- `seguridad.py` - JWT creation/verification, password hashing, user authentication
- `logger.py` - Structured logging setup (JSON format), request context helpers
- `rate_limiter.py` - SlowAPI rate limiting decorator and config
- `seeders.py` - Database seeding for development/testing

### Frontend: `frontend/src/`

**API Layer (`api/`):**
- `client.ts` - Axios instance with auth interceptor, token refresh, 401 retry logic
- `endpoints.ts` - API endpoint helper functions (organized by domain)

**State Management (`stores/`):**
- `authStore.ts` - Zustand store: token, user, tenants, tenant selection, impersonation

**Components (`components/`):**
Organized by feature:
- `layout/` - AppShell, Sidebar, BottomNav, MobileDrawer, LogoutButton
- `auth/` - RoleGuard (checks rol_en_tenant), SuperadminGuard
- `ui/` - Reusable: Modal, SearchInput, DataCard, Skeleton, WhatsAppButton, SpringTransition
- `unified/` - ProductInventoryModule (compound module combining products + inventory)
- `crm/` - KanbanBoard, DealDetailModal, ActivityFeed
- `soporte/` - TicketList, TicketDetail, CreateTicketModal
- `onboarding/` - OnboardingWizard, StepProgress
- `virtualized/` - VirtualList (react-window wrapper for 100k+ rows)

**Pages (`pages/`):**
Full-page route components (one per route):
- `LoginPage.tsx` - Login form (Clerk + legacy fallback)
- `RegistroPage.tsx` - Registration (Clerk + legacy fallback)
- `ClerkCallbackPage.tsx` - Clerk OAuth callback, JWT exchange
- `SelectTenantPage.tsx` - Multi-tenant picker (shown if user has >1 tenant)
- `EmpresaWizardPage.tsx` - Company onboarding wizard (shown after registration)
- `DashboardPage.tsx` - Home/analytics dashboard
- `ProductosPage.tsx`, `VentasPage.tsx`, `InventarioPage.tsx`, etc. - Domain pages
- `TenantsPage.tsx` - Superadmin: manage all tenants
- `ConfigPage.tsx` - Tenant configuration (settings, users, accounting)

**Hooks (`hooks/`):**
Custom React hooks (per-feature):
- `useAuth.ts` - Wrapper for auth store
- `useNavigation.ts` - Navigation helpers
- `useOnboarding.ts` - Onboarding flow state
- `useCursorPagination.ts` - Cursor-based pagination for lists
- `useSSE.ts` - Server-Sent Events subscription
- `useMediaQuery.ts` - Responsive design queries
- `useAnalytics.ts` - Page view tracking

**Types (`types/`):**
TypeScript type definitions:
- `index.ts` - All types (User, Tenant, Producto, Venta, etc.)

**Utilities (`utils/`):**
- Helper functions (formatting, validation, etc.)

**Assets (`assets/`):**
- Images, icons, fonts

## Naming Conventions

### Files

**Python:**
- Modules: `lowercase_with_underscores.py`
  - Routes: `rutas/auth.py`, `rutas/productos.py`
  - Services: `servicios/servicio_auth.py`, `servicios/servicio_productos.py`
  - Models: `datos/modelos.py`, `datos/modelos_tenant.py`
  - Utilities: `utils/seguridad.py`, `utils/logger.py`
- Classes: `PascalCase` (e.g., `ServicioProductos`, `Productos`)
- Functions: `snake_case` (e.g., `crear_producto()`, `obtener_tenant_por_id()`)
- Constants: `UPPER_SNAKE_CASE` (e.g., `API_PREFIX`, `DB_POOL_SIZE`)

**TypeScript/React:**
- Components: `PascalCase.tsx` (e.g., `ProductosPage.tsx`, `Modal.tsx`)
- Hooks: `camelCase.ts` prefixed with `use` (e.g., `useNavigation.ts`, `useAuth.ts`)
- Utilities: `camelCase.ts` (e.g., `formatPrice.ts`)
- Types: `PascalCase` in definition (e.g., `type User = { ... }`)
- Stores: `camelCase` + `Store.ts` (e.g., `authStore.ts`)
- Constants: `UPPER_SNAKE_CASE` (e.g., `API_BASE`, `STORAGE_KEY`)

### Function/Method Names

**Services (business operations, not CRUD):**
- `crear_*()` - Create and related setup
- `obtener_*()` - Fetch single entity
- `listar_*()` - Fetch multiple entities
- `actualizar_*()` - Update entity
- `eliminar_*()` - Soft delete entity
- `registrar_*()` - Record an event (audit, payment, etc.)
- `anular_*()` - Cancel/void document
- `generar_*()` - Generate report/document

**Routes (HTTP handler):**
- `crear_*()`, `obtener_*()`, `listar_*()`, etc.
- Return type: Pydantic DTO response class

**Hooks (React custom hooks):**
- `useAuth()`, `useNavigation()`, `useCursorPagination()`
- Return type: `[state, setState]` or object with methods

## Where to Add New Code

### New Feature/Domain (e.g., Suppliers Module)

1. **Database (if needed):**
   - Add model to `backend/app/datos/modelos.py` (or new `modelos_suppliers.py`)
   - Create migration: `alembic revision --autogenerate -m "add suppliers table"`
   - Add RLS policy to migration

2. **Backend API:**
   - Create service: `backend/app/servicios/servicio_suppliers.py` with service class
   - Create route: `backend/app/rutas/suppliers.py` with router
   - Add DTO schemas to `backend/app/datos/esquemas.py` (SupplierCreate, SupplierResponse, etc.)
   - Register router in `backend/app/main.py` line ~440

3. **Frontend:**
   - Create page: `frontend/src/pages/SuppliersPage.tsx`
   - Create API endpoints: Add to `frontend/src/api/endpoints.ts`
   - Add route: Register in `frontend/src/App.tsx` Router
   - Create components: `frontend/src/components/suppliers/` if needed
   - Add type: `type Supplier = {...}` to `frontend/src/types/index.ts`

### New Component/UI Module

1. **Location:** `frontend/src/components/{feature}/ComponentName.tsx`
2. **Props Interface:** Define `interface ComponentNameProps { ... }` in same file
3. **Custom Hook (if needed):** Extract state logic to `frontend/src/hooks/useComponentName.ts`
4. **Types:** Add to `frontend/src/types/index.ts` if shared across components

### Utilities/Helpers

**Backend:**
- Reusable functions: `backend/app/utils/`
- Shared model logic: `backend/app/datos/mixins.py` for common fields

**Frontend:**
- Pure functions: `frontend/src/utils/` with `camelCase.ts` naming
- React hooks: `frontend/src/hooks/useNameHere.ts`
- API endpoints: Add to `frontend/src/api/endpoints.ts`
