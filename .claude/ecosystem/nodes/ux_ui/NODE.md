## Operational Rules

1. **Design system is law** — deviations require UX Lead approval + documented reason
2. **Prototype before build** — no new feature enters development without clickable prototype
3. **Test with real users** — usability tests with ≥ 5 users per major flow
4. **Accessibility is not optional** — every component tested for keyboard, screen reader, contrast
5. **Design review per sprint** — attend squad demos, flag UX regressions
