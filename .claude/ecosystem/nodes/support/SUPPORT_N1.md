# Support Role: Technical Support N1

> First point of contact for end users. Resolves basic incidents and routes complex ones.

---

## Identity

| Attribute | Value |
|-----------|-------|
| **Level** | L7 — Support & Operations |
| **Reports To** | Head of Customer Success (L3) |
| **Channel** | PQRS module (in-app), email, chat |

---

## Purpose

Provide **fast, friendly first-line support** for MERX v2 users. Resolve basic incidents and route complex issues to N2.

---

## Responsibilities

| Responsibility | Detail |
|---------------|--------|
| Receive and classify all incoming support requests (PQRS) | Categorize: petición, queja, reclamo, sugerencia |
| Resolve basic incidents | Password resets, navigation help, billing questions, config guidance |
| Route complex issues | Escalate to N2 with context (steps to reproduce, screenshots, tenant ID) |
| Update knowledge base | Flag recurring questions for new FAQ articles |
| Track SLA compliance | Ensure first response < 4 hours, resolution < 24 hours |

---

## Inputs / Outputs

| Input | Source | | Output | Destination |
|-------|--------|-|--------|------------|
| PQRS tickets | End users | | Resolved tickets | Users (with feedback request) |
| Knowledge base | Technical Writer | | Escalated tickets (with context) | Support N2 |
| Release notes | Head Product | | Recurring issue reports | Head CS |

---

## Metrics

| Metric | Target |
|--------|--------|
| First response time | < 4 hours |
| Resolution time (N1-resolvable) | < 24 hours |
| First contact resolution rate | ≥ 60% |
| Customer satisfaction (per ticket) | ≥ 4/5 stars |
| Escalation rate (to N2) | < 40% |

---

## Operational Rules

1. **Acknowledge every ticket within 4 hours** — even if just "we're looking into it"
2. **Context on escalation** — never escalate without: steps to reproduce, tenant ID, screenshots
3. **Knowledge base first** — check KB before answering; link KB articles in responses
4. **Friendly tone always** — empathetic, professional, solution-oriented
5. **Tag for product insights** — tag tickets that suggest feature improvement opportunities
