## DECISIÓN 1 — Phase 2: IA Costeo

**DECISION LOCKED: claude-haiku-4-5 via Anthropic API**

### Opciones evaluadas

| Opción | Costo/mes (150K req) | Pros | Contras |
|--------|---------------------|------|---------|
| claude-haiku-4-5 (batch) | $0.83 | Mismo vendor, structured output nativo, prompt caching | Costo mayor que Gemini |
| claude-haiku-3-5 (batch) | $0.66 | Más económico | Menor performance que 4.5 |
| Gemini 2.0 Flash (batch) | $0.09 | 10× más económico | Nuevo vendor, nuevas credenciales, nuevo SDK |
| claude-sonnet-4-6 | $3.30 | Máxima calidad | Overkill para costeo, 4× más caro |

### Decisión

**claude-haiku-4-5** con prompt caching habilitado. Razón: consolidación de vendor (Merx ya usa Anthropic SDK), structured output via `tool_use` con schema Pydantic, latencia más baja del lineup Anthropic para respuestas síncronas. Costo proyectado: $0.83/mes al pico de 150K requests — bien por debajo del techo de $50/mes.

### Configuración técnica

- Model ID: `claude-haiku-4-5`
- Modo: synchronous (interacción usuario directa); batch opcional si se escala a >500K requests/mes
- Prompt caching: sistema prompt (~500 tokens) cacheable en cada request
- Output schema: `{precio_sugerido: Decimal, margen_esperado: Decimal, justificacion: str, alertas: list[str]}`
- Env var nueva: `ANTHROPIC_API_KEY` (agregar a Railway antes de Phase 2)
- Riesgo: Decimal serialization — todos los outputs numéricos del LLM deben castearse vía Pydantic antes de storage

### Deferred (OUT OF SCOPE)

- R1.3: Análisis predictivo flujo de caja — requiere historial transaccional suficiente
- R1.4: Chatbot WhatsApp para clientes del tenant — Phase 5+ backlog
- Gemini Flash como alternativa — re-evaluar si volumen supera 500K req/mes (10× más barato)

## DECISIÓN 4 — Phase 5: WhatsApp Business

**DECISION LOCKED: BSP (WATI o 360dialog) + número compartido Merx**

### Opciones evaluadas

| Opción | Setup time | Costo Colombia utility | Número por tenant |
|--------|-----------|----------------------|-------------------|
| BSP (WATI) | 24–48h | ~$0.001 (markup 20%) | Compartido o dedicado |
| BSP (360dialog) | 24–48h | ~$0.001 | Compartido o dedicado |
| Meta Cloud API directo | 2–10 días hábiles | $0.0008 | 1 WABA = 1 número |

### Decisión

**BSP (WATI o 360dialog)** para aceleración de onboarding WABA. **Número compartido Merx** para MVP — un WABA, nombre del tenant en el body del mensaje. Razón: BSP reduce verificación de 10 días a 24–48h. Número compartido elimina overhead de registrar WABA por tenant (bloqueante para micro-negocios sin NIT establecido).

### Configuración técnica

- Costo proyectado: $33/mes (100 tenants × 100 utility msgs + 20 marketing msgs)
- Templates requeridos: "factura_enviada" (utility), "pago_confirmado" (utility)
- Template approval time: 24–72h post-WABA verification
- Env var: `WHATSAPP_API_KEY`, `WHATSAPP_PHONE_NUMBER_ID`

### Open question (resolver antes de Phase 5)

- ¿Meta requiere NIT colombiano para business verification incluso via BSP? → BSP maneja compliance, reduce fricción.

### Deferred (OUT OF SCOPE)

- Número propio por tenant: re-evaluar en Phase 5+ cuando base de tenants crezca
- R2.3: Recepción de pedidos por WhatsApp — requiere flujo conversacional completo
